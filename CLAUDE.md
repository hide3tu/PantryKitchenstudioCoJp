# CLAUDE.md - テキストパントリー デザインガイドライン

## プロジェクト概要

- **サイト名**: テキストパントリー
- **URL**: https://pantry.kitchenstudio.co.jp/
- **事業内容**: クリエイティブ執筆・シナリオ制作（ゲームシナリオなど）
- **技術スタック**: Astro + Tailwind CSS v4
- **ページ構成**: ホーム、スタッフ紹介、仕事一覧（ポートフォリオ）、問い合わせ

---

## デザイン方針

### トーン: Editorial/Magazine（編集的・マガジン風）

- 洗練されたタイポグラフィを主役に
- コンテンツ（物語・執筆実績）が引き立つレイアウト
- 雑誌のような構成感と余白の美しさ

### カラーシステム: モノクローム

```css
:root {
  --background: oklch(1 0 0);           /* 白 */
  --foreground: oklch(0.1 0 0);         /* 黒に近い */
  --muted: oklch(0.95 0 0);             /* 薄いグレー（背景用） */
  --muted-foreground: oklch(0.4 0 0);   /* 中間グレー（補助テキスト） */
  --accent: oklch(0.3 0 0);             /* アクセント用グレー */
  --border: oklch(0.85 0 0);            /* ボーダー */
}
```

※ ライトモードのみ対応

### タイポグラフィ: ミックス

| 用途 | フォント | 特徴 |
|------|----------|------|
| 見出し | Noto Serif JP / Playfair Display | セリフ体・文学的な印象 |
| 本文 | Noto Sans JP / Geist | サンセリフ体・読みやすさ重視 |

メリハリのある読み心地を実現する。

## デザイン原則

### フロントエンドデザイン美学

> 参照: https://github.com/anthropics/claude-code/blob/main/plugins/frontend-design/skills/frontend-design/SKILL.md

#### デザイン思考の4要素

1. **目的（Purpose）** - このインターフェースは何を解決するか
2. **トーン（Tone）** - Editorial/Magazine を採用
3. **制約（Constraints）** - 技術・アクセシビリティ要件
4. **差別化（Differentiation）** - 何が記憶に残るか

#### 5つの美的原則

1. **タイポグラフィ** - 個性的で美しいフォント選択。見出しと本文のコントラスト
2. **色とテーマ** - CSS変数で一貫性を確保。モノクロームで洗練された印象
3. **モーション** - 戦略的なアニメーション（ページロード時のスタガー効果、ホバー状態の変化）
4. **空間構成** - 非対称性を恐れない、寛大なネガティブスペース
5. **背景・視覚詳細** - 深さと雰囲気の創造（グラデーション、微細なテクスチャ）

#### 避けるべき要素

- Inter, Roboto, Arial, Space Grotesk などのジェネリックフォント
- 白背景 + 紫グラデーション（AIっぽい印象）
- 予測可能なレイアウト・コンポーネントパターン
- 文脈固有のキャラクターが欠けた汎用的なデザイン

### UX心理学の活用

> 参照: https://www.shokasonjuku.com/ux-psychology

#### 主要な心理効果

| 効果 | 説明 | 応用 |
|------|------|------|
| 美的ユーザビリティ効果 | 美しいデザインは使いやすいと感じられる | ビジュアル品質を高める |
| 認知負荷の軽減 | 情報処理の精神的エネルギーを減らす | 情報を段階的に開示 |
| ドハティの閾値 | 0.4秒以内の応答時間が理想 | ページ速度の最適化 |

#### 行動促進の心理学

| 効果 | 説明 | 応用 |
|------|------|------|
| 段階的要請 | 小さな要求から始めると大きな要求も受け入れやすい | 問い合わせへの導線を段階的に |
| ナッジ効果 | 非強制的に最適な選択へ誘導 | CTAボタンの配置・デザイン |
| 好奇心ギャップ | 情報の欠如が興味を引く | キャッチコピーに適度な謎を |

#### その他の重要な概念

- **損失回避・希少性効果** - 限定感の演出
- **社会的証明** - 実績・クライアントの表示
- **ピーク・エンドの法則** - 最高の瞬間と終了時点が印象を決める

## 実装ガイドライン

### レイアウト

- 最大幅: 1200px（コンテンツエリア）
- 寛大な余白を使用
- グリッド破壊要素で視覚的な興味を生む

### アニメーション

- ページロード時: スタガー効果（要素が順番にフェードイン）
- ホバー: 控えめだが気持ちの良いフィードバック
- スクロールトリガー: 必要に応じて

### 画像

- WebP形式を推奨
- アスペクト比を統一（ポートフォリオなど）
- alt属性を必ず設定
